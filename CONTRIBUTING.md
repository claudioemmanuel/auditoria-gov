# Contributing to OpenWatch (Public Layer)

Thank you for your interest in contributing! We welcome contributions from everyone. By participating, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

---

## Ways to Contribute

- **Code**: Features, bugfixes, refactoring, tests (Python, TypeScript, Next.js, FastAPI)
- **Documentation**: Improve docs, guides, onboarding, or translations
- **Design**: UI/UX, accessibility, or branding
- **Community**: Issue triage, discussions, or outreach

---

## Getting Started

1. **Fork** this repository on GitHub
2. **Clone** your fork:
	```bash
	git clone https://github.com/your-username/openwatch.git
	cd openwatch
	```
3. **Create a branch** for your work:
	```bash
	git checkout -b my-feature-branch
	```
4. **Make your changes** (add tests/docs as needed)
5. **Commit and push**:
	```bash
	git add .
	git commit -m "Describe your changes"
	git push origin my-feature-branch
	```
6. **Open a Pull Request** on GitHub with a clear description and reference any related issues

---

## Contribution Guidelines

- Follow the [Code of Conduct](CODE_OF_CONDUCT.md)
- Write clear, descriptive commit messages
- Add/maintain tests for new features or bugfixes
- Update documentation as needed
- Be respectful and constructive in reviews/discussions

---

## Reporting Issues

If you find a bug or have a feature request, [open an issue](https://github.com/openwatch-br/openwatch/issues) with as much detail as possible.

---

## Code Style

- **Python**: [PEP8](https://www.python.org/dev/peps/pep-0008/), use type hints
- **JavaScript/TypeScript**: [StandardJS](https://standardjs.com/), [Prettier](https://prettier.io/)
